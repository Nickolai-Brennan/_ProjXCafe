# ☕ CAFE — Product Overview

## Core Idea

Cafe is a **profile-driven micro-task marketplace** where builders and users exchange small, clearly defined work units using a **credit-based system** instead of direct cash transactions.
The platform combines elements of freelance marketplaces, social platforms, and gamification into a single workflow.

---

## What the Product Does

Cafe enables users to:

• Post projects broken into small actionable tasks
• Let other users claim and complete those tasks
• Pay using platform credits (with optional tips)
• Approve work before releasing payment
• Build reputation, XP, and public proof of work

At the same time, it allows skilled users to:

• Discover quick tasks aligned with their skills
• Earn credits and build a public portfolio
• Increase visibility through activity and performance
• Progress through levels and reputation systems

---

## Core User Roles

### 1. Project Creator (Needs Work Done)

• Posts projects and tasks
• Defines requirements and rewards
• Reviews submissions
• Releases credits upon approval
• Gains XP for participation and management

---

### 2. Builder / Worker (Completes Tasks)

• Browses and filters available tasks
• Claims and completes work
• Submits deliverables
• Earns credits and XP
• Builds reputation and portfolio

---

## Key Product Pillars

### 1. Task-Based Marketplace

Work is broken into **small, fast, clearly scoped units**
Example:
• “Fix this React component”
• “Design a logo draft”
• “Write 300-word summary”

Focus: speed, clarity, low friction

---

### 2. Credit Economy (Controlled Payments)

Instead of direct payments:

• Users operate on **credits**
• Credits are **locked in escrow** when a task is claimed
• Released only after approval
• Tips provide optional upside

This reduces disputes and simplifies transactions

---

### 3. Profile as Identity + Resume

Each user has a **public, dynamic profile** that shows:

• Completed tasks
• Active projects
• XP level
• Reviews and ratings
• Skills and tools

Profiles function as both:
→ Resume
→ Social identity

---

### 4. XP + Gamification Layer

User actions generate XP:

• Posting projects
• Completing tasks
• Receiving high ratings

XP drives:
• Levels
• Badges
• Visibility
• Trust signals

This creates engagement beyond money

---

### 5. Social + Discovery Layer

Cafe is not just transactional—it’s visible and interactive:

• Activity feeds
• Trending tasks
• Skill-based discovery
• Follow system
• Achievement sharing

---

## Core Workflow

### Standard Flow

1. Creator posts a task
2. Worker claims task
3. Credits are locked
4. Worker submits deliverable
5. Creator reviews
6. Approval triggers credit release
7. XP is awarded to both parties

---

## Problem Being Solved

Traditional freelance platforms:

• Too slow for small tasks
• High friction for onboarding
• Poor trust signals for new users
• Payment disputes and delays

Cafe solves this by:

• Making tasks **small and fast**
• Using **pre-funded credit escrow**
• Creating **visible reputation systems**
• Adding **gamification to increase engagement**

Cafe =
**Fast, structured, social micro-work platform**

## One-Line Definition

Cafe is a **gamified, credit-based marketplace for completing small, real-world tasks with public proof of skill and reputation.**
